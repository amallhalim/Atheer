## üö¶ Getting Started

### 1. Reporting Bugs
- **Check for existing issues**: Before opening a new issue, please search the [issue tracker](https://github.com/your-username/atheer/issues) to see if it has already been reported.
- **Use the template**: When creating a new issue, provide as much detail as possible, including steps to reproduce, expected behavior, and screenshots if applicable.

### 2. Suggesting Enhancements
- **Open an issue**: Explain the feature you'd like to see, why it's useful, and how it fits into the "Glow & Glass" aesthetic.
- **Join the discussion**: We love to brainstorm! Community feedback helps shape the roadmap.

## üõ†Ô∏è Development Workflow

### 1. Environment Setup
Make sure you have Node.js 18+ installed.
```bash
git clone https://github.com/your-username/atheer.git
cd atheer
npm install
npm run dev
```

### 2. Branching Strategy
- **`main`**: The stable branch.
- **`feature/xxx`**: For new features.
- **`fix/xxx`**: For bug fixes.
- **`docs/xxx`**: For documentation updates.

### 3. Coding Standards
- **TypeScript**: All new code must be fully typed.
- **Tailwind**: Use theme variables (e.g., `text-primary`) instead of static hex codes.
- **Linting**: Run `npm run lint` before committing to ensure code quality.

---

## üì• Pull Request Process

1.  **Fork the repo** and create your branch from `main`.
2.  **Make your changes**, ensuring they follow the project's architecture (static-data vs. components).
3.  **Test your changes** locally in multiple themes.
4.  **Submit a PR**:
    - Describe the changes in detail.
    - Reference any related issues (e.g., `Closes #123`).
    - Include screenshots for UI changes.
5.  **Review**: A maintainer will review your PR and may suggest changes before merging.
